주문조회 시스템
판매자회원의 판매목록조회)
 seller ID를 FK로 하는 테이블을 만들어서 주문시 orderid/orderno/sellerid/itemlist/insert//
0. 무통장입금의 경우 admin에서 수동으로 결제완료를 해주어야 함.
1. 입금대기 | 결제완료->배송준비중->배송출발 | 주문취소
	입금대기: 무통장입금 한정 status
	결제완료: 결제만 완료되고 창고쪽으로는 주문정보가 안 넘어간 상태.
	배송준비중: 창고에 주문정보 넘어감.
	배송출발: 창고에서 송장 등록함.
	주문취소
추가로 판매자전용)미지급: 송장은 떴으나 금액이 판매자에게는 안들어옴
지급완료: 판매자에게 정산완료
	
회원등급별 페이지 차별점
admin: 	itemlist에서 삭제 버튼이 일괄적으로 다 뜸.
	wishlist, cart못봄
	orders에서 모든 회원의 주문내역을 볼 수 있음
	main에 관리자용 메뉴 존재: 회원 전체 목록, 다른 관리자 회원 추가, 

seller:	itemlist에서 자기가 올린 글에는 삭제 버튼이 뜸.
	wishlist에서 자기가 올린 상품 목록과 wishlist로 정해진 수<-이거 구현 안될거같아 그냥 못보게...
	cart 못봄
	orders에서 자기가 받은 주문내역이 뜸(이번달의 매출도 볼수있으면 좋을듯)
	
buyer:	itemlist에서 장바구니 담기, 위시리스트 담기 가능
	cart, wishlist 사용가능
	orders에서 자기가 한 주문내역이 뜸(주문단계 보임)

비회원:	 itemlist에서 장바구니에 담기 가능
	cart 사용가능
	orders에서 비회원 주문번호를 누르면 주문단계가 보임

필요한 order값
id: PK용 id
orderlist: String. item id값의 리스트. 구분자 _
buyername: 구매자 아이디
sellername: 판매자 아이디(X)
packagename: 주문자이름
address: 주소
postnumber: 우편번호
phonenumber: 전화번호
paymethod: 결제방법(무통장입금/ 카드)
price: 총 결제금액(배송비 제외?) <-그냥 배송비까지 합쳐서 해도 될거같아
orderno: 주문번호(비회원의 경우 이 값으로 주문목록에 접속해야함)
orderstate: 주문상태
packageno: 송장번호
